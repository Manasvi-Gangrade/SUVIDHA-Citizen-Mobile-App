1.3 What Has Been Built (MVP Scope) 
The current MVP demonstrates end-to-end citizen interaction flows across all three 
departments, including: 
• Touch-optimized multi-page kiosk UI (React.js) with language selection and secure 
authentication 
• Electricity Department: Bill payment, complaint registration, and new connection 
request with receipt generation 
• Gas Department: New connection application, leakage complaint, and KYC 
document upload 
• Municipal Corporation: Water supply complaints, waste management grievances, 
and civic request submission 
• Real-time complaint and application status tracking with unique reference IDs 
                                                                                          
• Admin dashboard with service analytics, kiosk health monitoring, and complaint 
resolution tracking 
• SMS notification integration for OTP, acknowledgements, and status updates

3. UI/UX Suitability for Touch-Based Kiosk Interface 
3.1 Touch-First Design Principles 
Yes, the SUVIDHA UI is specifically and exclusively designed for touch-based kiosk usage. 
The design addresses unique kiosk interaction constraints that conventional web 
applications do not consider.

3.3 UI Component Specifications 
• Button sizes: Minimum 64px height for primary actions, 48px for secondary actions 
• Font sizes: Minimum 16px for body text, 20px for labels, 28px for headings – scalable 
to large-font mode (1.5x) 
• Color palette: High-contrast scheme with clear visual hierarchy; WCAG AA compliant 
• Progress indicators: Step-by-step wizard with visual breadcrumbs showing current 
step and total steps 
• Feedback: Every touch interaction provides immediate visual feedback (color 
change, ripple effect) 
• Timeouts: 90-second inactivity warning with 30-second countdown before session 
reset 
• Error handling: Inline validation with clear, non-technical error messages and 
suggested corrections

4.3 Offline Mode Architecture 
SUVIDHA uses a local-first caching approach for continuity during network outages: 
                                                                                          
• Service Worker + IndexedDB: Form data and in-progress applications cached locally 
• Offline-capable features: Complaint registration, form filling, token/queue number 
generation, information display 
• Features requiring connectivity: Payment processing, OTP verification, real-time 
status updates, document upload 
• Sync mechanism: Automatic background sync when connectivity is restored; user 
notified of pending submissions 
• Conflict resolution: Timestamp-based last-write-wins with server-side validation on 
sync

6. Security Architecture & Design 
6.1 Security by Design Philosophy 
Security in SUVIDHA is not an afterthought — it is embedded as a foundational principle at 
every layer of the system architecture. The approach follows Defence in Depth, where 
multiple independent security controls protect citizen data and ensure transaction integrity 
even if a single layer is compromised. 
Layered Security Architecture (Defence in Depth)

6.3 Security Integration into System Design 
Security controls are designed into the architecture, not added after: 
• Secure SDLC: Security requirements defined at the design stage; code review 
checklists include OWASP Top 10 
• Secrets Management: API keys, database credentials, and JWT secrets stored in 
environment variables / vault; never hardcoded 
• Dependency Security: npm audit and automated vulnerability scanning in the CI/CD 
pipeline 
• Kiosk Hardening: Only whitelisted applications can run; OS firewall restricts outbound 
connections to approved backend endpoints only 
• Data Minimization: Only data necessary for the specific service transaction is 
collected and retained; no persistent citizen profiles without explicit consent 
• Session Isolation: Each citizen session is completely isolated; no data from previous 
sessions accessible after logout or timeout

6.5 Compliance Summary 
• Digital Personal Data Protection (DPDP) Act 2023: Explicit consent collection; 
minimal data retention; right to erasure support 
• Government IT Security Guidelines (CERT-In): Aligned with MeitY security 
framework for e-governance applications 
• PCI-DSS: Payment card data never stored; tokenized transactions through certified 
gateways 
• IT Act 2000 (Section 43A): Reasonable security practices for sensitive personal data 
• RTI Readiness: All administrative transactions logged and exportable in structured 
format for RTI compliance
